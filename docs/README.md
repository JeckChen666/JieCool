# 文档中心

本文档中心提供 JieCool 项目的完整文档体系，帮助开发者快速了解项目结构、API接口、开发规范等信息。

## 📁 文档目录结构

```
docs/
├── README.md              # 文档中心说明（本文档）
├── project.md             # 项目快速上手指南
├── api/                   # API接口文档
├── execute/               # 项目修改日志
├── front-web/             # 前端技术文档
├── server/                # 后端技术文档
├── modules/               # 功能模块设计文档
├── resource/              # 外部测试资源
├── depend/                # 依赖管理文档
├── db/                    # 数据库设计文档
└── attention/             # 开发注意事项
```

## 📖 各目录作用

### 🚀 快速开始
- **[project.md](project.md)** - 新程序员快速上手指南，包含环境准备、项目启动、核心功能等

### 🔌 API接口文档 (`api/`)
**作用**：记录所有后端暴露的接口，按功能模块划分

**文档标准**：
- 接口 URL
- 接口描述
- 传入参数和注释
- 响应参数和注释
- 追求简单快速易懂

**接口模块**：
- `README.md` - API接口概览
- `auth.md` - 认证模块接口
- `file_api.md` - 文件管理接口
- `daily.md` - 每日一句接口
- `config.md` - 动态配置接口
- `visit.md` - 访问统计接口

### 📝 项目修改日志 (`execute/`)
**作用**：简要的项目修改日志，以日期作为划分

**命名规范**：`YYYY-MM-DD-功能描述.md`

**内容类型**：
- 功能开发记录
- 问题修复记录
- 架构调整记录
- 性能优化记录

### 🎨 前端技术文档 (`front-web/`)
**作用**：存放前端相关组件的文档，来自开源社区

**包含内容**：
- `arco-design_doc/` - Arco Design 组件库文档
- `nextjs_doc/` - Next.js 框架文档

### ⚙️ 后端技术文档 (`server/`)
**作用**：存放后端相关的组件文档，来自开源社区

**包含内容**：
- `goframe_doc/` - GoFrame 框架文档

### 🧩 功能模块设计 (`modules/`)
**作用**：项目中模块设计的描述，以模块划分

**模块文档**：
- `MODULES_README.md` - 模块总览
- `auth.md` - 认证模块设计
- 其他功能模块文档...

### 🧪 外部测试资源 (`resource/`)
**作用**：项目测试过程中用于测试的外部测试资源

**包含内容**：
- `iciba.md` - 金山词霸API使用说明
- `test.png` - 测试图片资源
- 其他测试相关资源...

### 📦 依赖管理 (`depend/`)
**作用**：项目依赖的来源、版本与管理策略

**包含内容**：
- `index.md` - 依赖管理总览
- 前端和后端依赖版本信息
- 更新策略和安全考虑

### ⚠️ 开发注意事项 (`attention/`)
**作用**：跨团队需要关注的注意事项和通用规范

**包含内容**：
- `README.md` - 注意事项总览
- 代码规范要求
- 开发流程规范

## 🎯 文档使用指南

### 新手入门路线
1. **阅读** `project.md` - 快速了解项目并启动开发环境
2. **查看** `api/README.md` - 了解后端接口概览
3. **参考** `depend/index.md` - 了解项目依赖
4. **学习** `modules/` - 了解各功能模块设计

### 开发过程查阅
- **API开发** → 查阅 `api/` 目录下对应模块文档
- **前端开发** → 查阅 `front-web/` 框架文档
- **后端开发** → 查阅 `server/` 框架文档
- **数据库设计** → 查阅 `db/` 文档
- **问题修复** → 查阅 `execute/` 历史记录

### 文档维护原则
- **API变更** → 及时更新 `api/` 目录对应文档
- **功能开发** → 更新 `modules/` 目录模块文档
- **问题修复** → 在 `execute/` 目录添加日期记录
- **依赖变更** → 更新 `depend/` 目录版本信息
- **架构调整** → 更新相应的设计文档

## 📝 文档编写规范

### API文档规范
- 统一的接口描述格式
- 清晰的参数说明和注释
- 完整的响应示例
- 错误码和异常处理说明

### 日志文档规范
- 按日期命名的文件
- 简洁明了的问题描述
- 清晰的解决方案
- 影响范围说明

### 模块文档规范
- 模块功能概述
- 设计思路说明
- 接口定义和调用方式
- 使用示例和注意事项
