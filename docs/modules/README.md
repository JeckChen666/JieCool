## 📖 文档编写规范

### 文档结构标准
每个模块文档严格遵循以下结构：

1. **模块预期实现的目标** - 设计目标和适用场景
2. **模块预期的功能点** - 详细功能列表和分组
3. **数据流向与处理逻辑** - 完整的数据流程图
4. **重点代码设计逻辑** - 核心算法伪代码实现
5. **模块功能使用方式** - 前后端集成和API调用
6. **第三方组件与数据库设计** - 技术栈和数据表结构

### 技术要求
- **语言无关的伪代码**：展示核心算法逻辑
- **数据驱动的设计**：基于真实代码的技术细节
- **完整的技术架构**：涵盖前端、后端、数据库全栈
- **实际代码对应**：所有设计都有相应的代码实现

## 🛠 技术栈总览

### 前端技术栈
- **框架**：Next.js 14.2.15 (App Router)
- **语言**：TypeScript
- **UI库**：Arco Design 2.66.5
- **状态管理**：React Hooks + Alova
- **样式方案**：Tailwind CSS

### 后端技术栈
- **框架**：GoFrame v2.9.3
- **语言**：Go 1.23.0
- **数据库**：PostgreSQL 18
- **ORM**：GoFrame ORM
- **认证**：JWT (HS256)

### 核心特性
- **前后端分离**：标准RESTful API设计
- **无状态认证**：JWT Token认证机制
- **动态配置**：运行时配置热更新
- **实时统计**：WebSocket实时数据推送
- **多媒体支持**：图片、视频、文件处理
