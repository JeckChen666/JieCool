# 执行日志目录说明

本目录用于记录项目关键操作与运行过程的执行日志，便于复现与审计。建议每次较大改动或一次完整的搭建/调试流程，都新增一份当天日志文件。

命名规范：
- 按日期创建：`YYYY-MM-DD-exec.md`
- 如同一天有多次独立流程，可按序号区分：`YYYY-MM-DD-exec-1.md`、`YYYY-MM-DD-exec-2.md`

日志内容建议包含：
- 操作目标与背景（为什么做、解决什么问题）
- 执行命令、关键参数与所在目录（如 npm、gf CLI）
- 结果与截图/URL（如本地预览地址、接口列表）
- 涉及的配置与代码文件（相对路径、文件名、要点）
- 问题与修复（报错信息、修复方法、代码变更摘要）
- 后续 TODO（需跟进事项）

注意事项：
- 不强制记录精确时间戳，但需按步骤清晰描述与可复现。
- 禁止泄露敏感信息（密码、密钥）；涉及连接参数时用占位符替换。
- 与 `docs/project.md`、`docs/depend/index.md` 保持一致性，变更后同步更新文档。
